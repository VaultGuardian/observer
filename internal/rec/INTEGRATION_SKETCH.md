# REC Integration Sketch

How Response Evidence Capture wires into the existing Observer pipeline.

## Critical Rules

1. **REC is enrichment-only in Phase 1.** It NEVER changes VerdictDeny, VerdictAlert, or any classification. It ONLY attaches evidence to alerts that were already generated.
2. **Dual-gate rule** is enforced at construction time in `liveCollector.Lookup()`, not via a getter method. `SafeBodyPreview` is exported for JSON serialization.
3. **Status code is a hard filter** in correlation, not a soft confidence downgrade.

## main.go Wiring

```go
// After config parsing:
recCfg := rec.DefaultCollectorConfig()
recCfg.Enabled = os.Getenv("REC_ENABLED") == "true"
if iface := os.Getenv("REC_INTERFACE"); iface != "" {
    recCfg.Interface = iface
}
collector := rec.NewCollector(recCfg)

// Start REC capture (non-blocking, non-fatal)
if err := collector.Start(ctx); err != nil {
    log.Printf("[observer] REC failed to start: %v (continuing without evidence capture)", err)
}
```

## Alert Building (after classification returns VerdictAlert or VerdictDeny)

```go
// EXISTING (unchanged):
alert := buildAlert(evt, result)

// NEW: Enrich with response evidence
evidence := collector.Lookup(rec.LookupRequest{
    Method:          extractMethod(normalizedLine),
    Path:            extractRawPath(normalizedLine), // RAW path with query string!
    Host:            extractHost(normalizedLine),
    UserAgent:       extractUserAgent(rawLine), // from original line, not normalized
    SourceContainer: evt.SourceName,
    StatusCode:      extractStatusCode(normalizedLine),
    Timestamp:       evt.Timestamp,
})

// Log evidence status in journal (ALWAYS, even when unavailable)
log.Printf("[ALERT] EventID=%s %s", evt.ID, evidence.ForJournal())

// Attach to alert struct
alert.Evidence = evidence
```

## Phase 1 Limitations

1. **PROXY BLIND SPOT:** REC captures plaintext HTTP between reverse proxy and backend. Responses generated by nginx/caddy/traefik directly (403s, 404s, redirects, static files) are NOT visible. Phase 1 returns `not_available_no_match` — it can't distinguish this from genuinely missing evidence.

2. **ENRICHMENT-ONLY:** REC does not influence classification or alert routing. An alert fires from log classification; REC tells you what the server returned.

3. **PLAINTEXT ONLY:** mTLS between proxy and backend = invisible to Phase 1. Phase 2 (eBPF) addresses this.

4. **SINGLE-HOST ONLY:** Captures on Observer's host network interfaces. Can't see responses from other hosts.

5. **PARTIAL BODY CLASSIFICATION:** Format detection runs on the truncated 2KB preview, not the full body.
